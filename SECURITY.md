# Security Policy

Reporting a vulnerability
- Please use GitHub’s “Report a vulnerability” (Security Advisories) on the repository page
- Provide steps to reproduce and any logs or screenshots that help explain impact

Scope
- This lab runs Elasticsearch without security enabled for simplicity; do not expose lab containers to untrusted networks
- Security features (xpack.security, TLS) should be enabled for any non-lab usage

Response
- We review reports as they are submitted and will acknowledge receipt and planned actions
